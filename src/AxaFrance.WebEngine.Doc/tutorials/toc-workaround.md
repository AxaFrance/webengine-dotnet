# Test Approaches and Tutorials

WebEngine Framework supports multiple test approaches depending on your team's methodology and project complexity.
This section presents step-by-step tutorials for each approach, with code samples in C#/.NET and Java.

| Approach | Description |
|---|---|
| **Linear Scripting** | Simple test scripts written directly in a unit test framework (e.g., MsTest, NUnit, JUnit). Best for simple or exploratory tests. |
| **Gherkin / BDD** | Test scenarios written in Gherkin (Given/When/Then) with SpecFlow (.NET) or Cucumber (Java). Best for feature-level acceptance tests. |
| **Keyword-Driven** | Tests modeled as a sequence of reusable keywords (actions), each with associated test data. Best for end-to-end test automation. |
| **Data-Driven** | Extends the keyword-driven approach with externalized test data (Excel + XML). Allows running the same test with multiple data sets without changing the code. |
